Если в репозитории есть важные данные, то их нужно сохранить, иначе при перезаписи они будут утрачены!

Сохранение данных в другую ветку:

git checkout master

git branch new-branch-to-save-current-commits


Обновление:


git fetch --all

git reset --hard origin/master


или (для других веток)


git reset --hard origin/<branch_name>


git reset обновляет ветку на указанный коммит, опция --hard обновляет все файлы в рабочей директории в соответствии с состоянием репозитория.
